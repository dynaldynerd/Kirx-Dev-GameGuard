================================================================================
 PROTECTION SYSTEM - INTEGRATION GUIDE
 For RF-Online WorldServer Source Code
================================================================================

OVERVIEW:
This Protection System recreates the fixes from 6.23_Guard 2 (SnooperGS)
directly into the WorldServer C++ source code. No external DLL needed.

FILES:
  Protection/
    ProtectConfig.h/.cpp    - INI config reader
    AntiHack.h/.cpp         - Speed/Wall/Fly/Invis hack detection
    AntiDupe.h/.cpp         - Trade/Trunk/Post/UMT dupe protection
    GameFixes.h/.cpp        - All game logic fixes
    ProtectionSystem.h/.cpp - Main manager (ties everything together)
    Configs/                - Default config files (copy to server folder)

================================================================================
 STEP 1: ADD FILES TO PROJECT
================================================================================

Add all .h and .cpp files from the Protection/ folder to your
WorldServer.vcxproj. Make sure the include path allows:
  #include "Protection/ProtectionSystem.h"

If using Visual Studio, right-click project -> Add -> Existing Item.

================================================================================
 STEP 2: INITIALIZE ON SERVER START
================================================================================

File: Entities/CMainThread.cpp
Function: CMainThread initialization (after database/map loading)

Add:
  #include "Protection/ProtectionSystem.h"

In the init function (after maps and tables are loaded):
  // Initialize Protection System
  char configPath[260];
  sprintf(configPath, "%s\\PROTECT\\Configs", GetServerPath());
  ProtectionSystem::Instance().Initialize(configPath);

================================================================================
 STEP 3: HOOK PLAYER ENTER/LEAVE WORLD
================================================================================

File: Entities/CMainThread.cpp or wherever player enter_world is handled

On player entering world (after character data is loaded):
  ProtectionSystem::Instance().OnPlayerEnterWorld(pPlayer);

On player disconnecting / leaving world:
  ProtectionSystem::Instance().OnPlayerLeaveWorld(pPlayer);

================================================================================
 STEP 21: HOOK PACKET RECEIVE (PACKET FLOOD DETECTION)
================================================================================

File: Entities/CNetworkEX.cpp or main packet dispatch
Function: Where incoming packets are dispatched

Before processing any packet from a player:
  #include "Protection/AntiCheat.h"

  if (!AntiCheat::Instance().OnPacketReceived(pPlayer->m_dwSerial))
  {
      // Packet flood detected - drop packet
      // Check if we should kick
      auto action = AntiCheat::Instance().EvaluatePlayer(pPlayer->m_dwSerial);
      if (action == AntiCheat::ACTION_KICK || action == AntiCheat::ACTION_BAN)
          pPlayer->Disconnect();
      return;
  }

================================================================================
 STEP 22: HOOK HWID REGISTRATION (LOGIN)
================================================================================

File: Where client sends HWID (login/handshake packet)

On receiving HWID from client:
  #include "Protection/AntiCheat.h"

  // Check if HWID is blocked
  if (AntiCheat::Instance().IsHWIDBlocked(hwid))
  {
      // Reject connection
      pPlayer->SendMsg_LoginResult(ERR_HWID_BLOCKED);
      pPlayer->Disconnect();
      return;
  }

  // Register HWID for tracking
  AntiCheat::Instance().RegisterHWID(pPlayer->m_dwSerial, hwid);

 STEP 23: HOOK SQL INPUT SANITIZATION
================================================================================

File: ALL packet handlers that accept string input from client

  #include "Protection/InputSanitizer.h"

  {
      // Invalid name or SQL injection attempt
      return;
  }

Guild creation:

      return;

Chat messages:
  std::string safeMsg = InputSanitizer::Instance().SanitizeChat(rawMessage);

Any string going into SQL query:
  std::string safeStr = InputSanitizer::Instance().SanitizeForSQL(userInput);
  // OR better: use parameterized queries

================================================================================
 NOTES
================================================================================

1. All fixes use CPlayer fields like m_dwSerial, m_fPosX, m_fPosY, m_fPosZ,
   m_nLevel, m_byRace, m_byGMGrade. Verify these field names match your
   WorldServer version. The RF-Online source in this repo uses these names.

2. Some fixes have placeholder comments (marked "Integration:") where you
   need to call specific WorldServer functions. This is because the exact
   function signatures vary between server versions.

3. Config files go in: <ServerPath>/PROTECT/Configs/
   Copy the defaults from Protection/Configs/ folder.

4. Log files are created in the server's working directory:
   - Protection_System.log
   - Protection_AntiHack.log
   - Protection_AntiDupe.log
   - Protection_GameFixes.log
   - Protection_InputSanitizer.log
   - Protection_ItemIntegrity.log
   - Protection_CombatIntegrity.log
   - Protection_AntiCheat.log

5. The system is designed to be non-destructive: if initialization fails
   or configs are missing, it uses safe defaults. No crash risk.

6. Thread safety: All modules use std::mutex for thread-safe access.
   The WorldServer is multi-threaded (network threads + main thread).

7. Total modules: 7
   - AntiHack       (6.23_Guard 2): speed/wall/fly/invis hack
   - AntiDupe       (6.23_Guard 2): trade/trunk/post/UMT dupe
   - GameFixes      (6.23_Guard 2): tower/guild/vote/potion/combine fixes
   - InputSanitizer (NEW): SQL injection, buffer overflow, name validation
   - ItemIntegrity  (NEW): premium item, set item effect, ore grade, embellish
   - CombatIntegrity(NEW): unit/tower/trap WH/FH, teleport, combat range
   - AntiCheat      (NEW): cheat scoring, packet rate limit, HWID block
